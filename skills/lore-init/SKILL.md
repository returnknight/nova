# 资料库初始化

你负责和作者一起把一个新故事从零整理成 Nova 可用的基础设定。这个流程同时服务 IDE 写作模式和互动模式。

## 工作流程

1. 先确认当前资料库为空或用户明确要求初始化。
2. 如果信息不足，先追问，不要直接写入。至少确认：
   - 题材、时代/世界类型和核心卖点
   - 主角或核心视角角色
   - 核心冲突、主要压力和故事目标
   - 世界规则、能力体系、禁忌或关键限制
   - 叙事风格、视角、篇幅偏好和内容边界
   - 互动模式需要的开局处境、初始地点、可行动空间
3. 将讨论结果整理成一版“待确认方案”，明确列出：
   - 将创建哪些资料库条目
   - `CREATOR.md` 将写入哪些创作规则
   - 哪些内容暂不确定，后续再补
4. 只有作者明确确认“确认、写入、保存、就按这个初始化、可以落库”等意图后，才执行写入。
5. 写入时：
   - 使用 `write_lore_items` 创建角色、世界观、地点、势力、规则、物品等长期稳定设定
   - 每个资料条目的 `brief_description` 必须以“类型 名称。”开头，后接 3-5 句身份、别名、关键事实、适用场景和触发词说明，最后写明“上下文出现相关内容时，一定要参考本项详情。”
   - 使用文件写入工具更新 `CREATOR.md`
   - 不写 `brainstorm.md`
   - 不写 `setting/outline.md`、`setting/progress.md`、`setting/character-states.md`
   - 不创建 `chapters/` 文件
   - 不创建互动 story，不伪造首个互动回合

## 资料库条目边界

- `character`：身份、人设、背景、核心目标、长期关系、能力边界
- `world`：世界类型、时代背景、基本秩序、关键常识
- `location`：长期会反复出现的地点和规则
- `faction`：组织、势力、阵营和利益关系
- `rule`：能力体系、世界硬规则、禁忌、代价、互动裁定原则
- `item`：关键物品、资源、道具和长期线索
- `other`：无法归类但需要长期复用的设定

资料库不要写章节规划、未来剧情详细安排、当前伤势/位置/持有物等短期状态。

## CREATOR.md 边界

`CREATOR.md` 只写全书长期创作规则，例如：

- 故事定位和目标读者
- 叙事视角、文风、对白风格
- 篇幅、节奏和章节偏好
- 禁止内容、尺度和雷区
- 互动模式开局基调和裁定偏好
- 作者已经确认的长期写作约束

不要把资料库条目的完整正文复制进 `CREATOR.md`。
