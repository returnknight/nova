# Nova

Nova is an AI writing workspace for long-form web novel creation. This context captures the product language used to describe the author's creative workflow.

## Language

**创作灵感**:
作品成型前的原始想法集合，包含题材、卖点、目标读者、核心设定、风格偏好和参考作品。创作灵感用于孵化大纲，不直接承担章节执行职责。
_Avoid_: 大纲草稿, 章节规划

**大纲**:
小说的长期结构承诺，描述主线、卷章安排、阶段目标、重大转折和章节方向。大纲低频变化，只有作者重构故事结构时才应调整。
_Avoid_: 长期细纲, 全书细纲

**章节组**:
一组连续章节形成的短期写作单元，用来承载一个相对完整的情节推进、连续冲突或阶段性目标。章节组由情节单元决定，而不是固定章数；系统可以建议 3-8 章作为常见范围。
_Avoid_: 单章任务, 固定章节批次, 固定章数

**细纲**:
章节组的写作执行方案，描述接下来这一组章节如何落笔、如何承接前后文、如何分配冲突、爽点、信息揭示和结尾钩子。细纲高频迭代，服务草稿生成与定稿，不替代大纲；细纲是作者可见的作品资产，不应一次性批量生成很多组。下一组细纲应由当前章节定稿进度和长期大纲共同驱动，不能只按旧大纲机械展开。
_Avoid_: 更详细的大纲, 单章大纲

**章节草稿**:
基于章节组细纲逐章生成或修改的候选正文。章节草稿用于作者审阅和迭代，不应自动成为后续故事连续性的事实来源；组内连续试写可以读取前序草稿作为候选上下文。
_Avoid_: 未完成正文, 自动正史

**章节定稿**:
作者确认纳入小说连续性的单章正文。章节定稿才是后续章节默认承接的正文事实，并在完成后推动进度和角色状态沉淀。
_Avoid_: 字数达标章节, 最后一次草稿

**草稿流程**:
可选的章节正文迭代流程，允许作者先生成和修订章节草稿，再确认进入章节定稿。高频日更作者可以跳过草稿流程，直接产出章节定稿；Nova 默认关闭草稿流程。
_Avoid_: 强制草稿模式, 唯一写作路径
